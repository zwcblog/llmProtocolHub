# LLM Protocol Hub

> 把 AI 厂商与协议的官方入口，重新按人能理解的方式编排一遍。

这不是一套重新发明的 AI 文档，也不是又一份很快会过时的参数摘录。

模型平台变化太快，二手整理总会落后；而官方文档本身，又往往分散在官网、开发者文档、API Reference、SDK 仓库、控制台与变更日志之间。真正消耗时间的，常常不是“看不懂”，而是“找不到”——或者更准确地说，找不到那个真正该看的入口。

**LLM Protocol Hub** 想解决的，就是这个问题。

它不试图替代官网，只做三件事：

- 把值得抵达的官方入口整理清楚
- 用统一结构标出厂商与协议的边界
- 在最少解释下，帮助你更快完成比较、定位与跳转

---

## 你会在这里看到什么

这个站点主要收录两类内容：

### 1. 厂商索引

围绕各家模型平台，整理最常用的官方入口，例如：

- 官网
- 官方文档
- API Reference
- SDK / GitHub
- Changelog / Release Notes
- Console / Platform

必要时，会补上一点极短的结构化说明，用来提示：

- 主 API 形态是什么
- 是否存在 OpenAI Compatible 兼容层
- 工具调用、流式、多模态的官方入口在哪里
- 与其他平台迁移时最容易误判的边界是什么

### 2. 协议索引

围绕跨厂商协议与事实标准，整理：

- 官方规范
- 参考实现
- 核心对象
- 关键流程
- 与相邻协议的边界

这里不会展开成长篇教程，只做足够克制的导航与摘要。

---

## 它不打算做什么

为了让这个站点长期可维护，它刻意不做以下几类事情：

- 不重写整套官方文档
- 不维护细碎且易过时的参数手册
- 不把入口站扩写成百科全书
- 不用大量二手描述替代一手来源

如果某个字段、能力或行为的最新定义与这里不一致，**以官方文档为准**。

---

## 阅读方式

你可以按三种方式进入：

- **按厂商查找**：当你已经知道目标平台，只是记不起入口分布
- **按协议查找**：当你在确认 MCP、A2A、Tool Calling、多模态或流式结构的官方说明
- **按索引跳转**：当你只是想尽快抵达原始出处

## 项目原则

这个项目坚持几条朴素的原则：

- 只收录官方来源
- 以链接为主，以解释为辅
- 页面尽量短、准、稳
- 优先帮助用户抵达原始出处，而不是停留在这里

如果把官方文档比作馆藏，这里做的事情，只是替它编一份更耐用的目录。
