# 维护规范

这个站点的目标不是“写完所有说明”，而是“把官方入口整理得足够好”。

## 核心原则

- 只收录官方来源
- 优先放链接，少做二次解释
- 页面尽量短、准、稳
- 让用户在 10 秒内点进官网

## 每页最低标准

无论是厂商页还是协议页，优先补这些：

- 官网
- 官方文档
- API Reference
- SDK / GitHub
- Changelog / Release Notes
- 控制台 / 平台入口（如有）

## 内容边界

建议做：

- 官方入口汇总
- 必要的一句话说明
- 页面间清晰跳转

不建议做：

- 大段复制官方文档
- 维护详细参数解释
- 过长的兼容性分析
- 把索引站写成百科站

## 写法建议

- 标题简洁
- 描述克制
- 少形容词，多链接
- 先给用户“去哪里看”，再决定“要不要解释”

## 设计原则

- 留白比堆叠更高级
- 导航比文字更重要
- 页面应该像目录，而不是像论文
- 稳定、耐看、长期可维护，比炫技更重要

## 维护节奏

- 新增厂商或协议时，先补官网与文档入口
- 发现链接失效时，优先修链接
- 如无必要，不扩写成长篇内容
